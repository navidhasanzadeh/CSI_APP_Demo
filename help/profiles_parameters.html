Profiles & Parameters (General)

Profiles are named presets stored by the application. Each profile keeps a group of parameters together so you can reuse them across experiments.

Common profile actions

  • New creates a fresh profile with default values.
  • Duplicate makes a copy you can tweak without losing the original.
  • Delete removes the profile from the list.
  • Save writes changes to disk so they are available next time.

Example

Create an "Lab_A_Normal" experiment profile for standard timing, then duplicate it as "Lab_A_Slow" to use longer baseline and action times. That way you can quickly switch between protocols without re-entering every field.
