Nexmon Framework & CSI Collection

CSI (Channel State Information) captures how Wi-Fi signals propagate between a transmitter and a receiver. The Nexmon framework enables CSI extraction from supported Broadcom chipsets by adding firmware patches and tooling that expose per-packet channel measurements.

What is Nexmon?

Nexmon is a firmware patching framework for Broadcom Wi-Fi chips. It provides the tooling and custom firmware needed to export CSI data from the radio interface. In this project, Nexmon-based sniffers produce PCAP files that include a Nexmon header and raw CSI payloads, which the application parses and stores.

How CSI is collected

  1. Configure the Wi-Fi profile: select the Nexmon framework and define your access point/sniffer devices, channels, bandwidth, and capture duration.
  2. Start the experiment: the app triggers the sniffer to capture packets per action window, saving PCAPs in the results folder.
  3. Extract CSI: the PCAP reader loads the Nexmon packets and converts the raw payload into CSI tensors for analysis and plotting.

What information CSI includes

  • Complex CSI per subcarrier: magnitude and phase values across OFDM subcarriers, represented as complex numbers.
  • Spatial streams: measurements per transmit/receive antenna pair when multiple antennas are enabled.
  • Timing metadata: per-packet timestamps used to align CSI with actions and other sensors.
  • Radio metadata: channel index, bandwidth, MCS, and other PHY details included in the capture headers.
  • Device metadata: MAC addresses and packet sequence information to identify the transmitter/receiver and filter traffic.

Tips for high-quality captures

  • Match the sniffer channel/bandwidth to the router configuration.
  • Minimize interference by choosing a clean channel and stable antenna placement.
  • Use packet counters to validate that data is flowing before recording a session.
  • Keep capture durations aligned with the experiment profile to avoid truncated data.

Common questions

  • Why is CSI missing? confirm the sniffer is running Nexmon firmware and the framework is set to Nexmon in the Wi-Fi profile.
  • Why are dimensions unexpected? the CSI tensor shape varies by number of spatial streams and bandwidth; single-stream captures omit the Tx axis.
