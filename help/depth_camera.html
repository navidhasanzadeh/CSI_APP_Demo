Depth Camera (Intel RealSense D455)

The depth camera integration lets you capture synchronized RGB + depth streams alongside CSI. The UI communicates with a local REST API provided by the D455 capture service and stores recordings per experiment action.

What the depth camera workflow provides

  • RGB + depth video captured per action window.
  • Depth timestamps that align with experiment actions.
  • Optional raw depth storage (NPZ) for offline processing.

Prerequisites

  • Hardware: Intel RealSense D455 connected over USB.
  • Depth API service: the D455 capture service running locally or on a VM. The UI polls its status endpoint and posts capture requests.
  • Network access: the API host must be reachable from the collection app.

Configuration in the app

  1. Open the Depth Camera tab in the Configuration window.
  2. Select or create a depth camera profile.
  3. Enable Depth camera to activate status polling.
  4. Set the API IP / host to the address where the D455 service runs. Include a port if it is non-default (for example, 10.211.55.8:5000).
  5. Choose FPS and RGB/depth resolutions that match your capture needs.
  6. Enable Save raw NPZ files if you want per-frame depth matrices. Use Save location to override the default recordings path.

Recommended capture workflow

  1. Start the D455 capture service and verify its status endpoint responds.
  2. In the app, click Refresh now to confirm the status summary.
  3. Run the experiment as usual; depth capture starts automatically for each action.
  4. After the session, check the recordings folder for RGB video, depth video, and timestamp CSV files.

Platform notes

Windows / Linux

Install Intel RealSense drivers and run the provided depth camera service. Ensure the API host and port are reachable from the UI machine.

macOS

RealSense drivers are not supported directly on macOS. Use a Windows or Linux VM that has USB passthrough for the D455. Run the capture service inside the VM and expose the API port to the host (for example using Parallels or VMware port forwarding).

Troubleshooting checklist

  • Status says disabled: confirm the depth camera checkbox is enabled.
  • Status error or timeout: verify the API IP/host and port, and check that firewalls or VM NAT settings allow access.
  • Blank recordings: confirm the D455 is detected by the service and that RGB/depth resolutions are supported by the device.
  • Large files: reduce FPS or choose a lower resolution before recording.
  • Raw NPZ storage missing: ensure Save raw NPZ files is enabled and the save directory exists.
