UI, Camera, Voice, and Time Settings

These profiles control how the interface behaves during the experiment, plus any camera overlays or voice prompts you want to enable.

Settings at a glance

  • UI profile: enable action counters, adjust fonts for readability, and show/hide countdown timers.
  • Camera profile: select the webcam device, enable hand recognition overlays, and confirm video resolution.
  • Voice profile: choose a language for spoken prompts and toggle the assistant on/off.
  • Time profile: sync to a preferred time server for consistent timestamps.

Depth camera notes (Intel RealSense D455)

The depth camera workflow expects a local webserver to stream the D455 feed so the UI can embed it alongside CSI capture. Ensure the webserver is running on the same machine and that the configured URL is reachable before starting the experiment.

On macOS, the RealSense drivers are not supported directly. We recommend using a Windows or Linux virtual machine (for example via Parallels) that can access the D455 USB device, run the depth camera webserver inside the VM, and expose it to the host through a forwarded port.

Example setup

For a demo session, enable voice prompts in English, use the built-in laptop camera, enable a large on-screen countdown timer, and verify the time profile is synchronized before collecting data.

Common issues

  • No camera feed: confirm OS permissions, device selection, and that the webcam is not locked by another app.
  • Depth camera blank: verify the local webserver URL and that the D455 is visible inside the VM (macOS).
  • Voice prompts missing: check the voice profile toggle and ensure gTTS and audio playback are available.
  • Time drift: re-sync the time profile and confirm your time server is reachable.
