Wi-Fi Capture & PCAP Management

Wi-Fi profiles define how CSI data is captured and how access points are managed.

Key parameters

  • Framework/type - which capture pipeline you are using.
  • Access points - routers, sniffers, and their login details.
  • Delete previous PCAPs - clear storage before a new run.
  • Count packets - enable packet counting for quick health checks.
  • Channel/bandwidth - align sniffer settings with the router configuration.
  • Capture duration - confirm action and baseline windows match the experiment profile.

Example

Before starting a new session, click Test Connections to verify each access point is reachable. Use Delete PCAPs if you need a clean slate. During the experiment, PCAPs are captured per action and stored in the results folder.

Common issues

  • Missing packets: check the sniffer channel, antenna orientation, and that the router is transmitting.
  • Login failures: re-validate router credentials and network connectivity.
  • PCAPs too large: reduce capture duration or disable unused antenna pairs.
  • Packet counter is zero: verify the MAC address filters and confirm the transmitter is active.
